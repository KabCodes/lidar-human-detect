# Unitree L2 LiDAR — Real-Time Human Detection

A real-time 3D human detection and tracking system using the **Unitree L2 4D LiDAR**, built entirely in Python with no ROS dependency. The system receives raw UDP packets from the LiDAR, parses them into a 3D point cloud, subtracts a pre-scanned background, and runs a three-classifier voting algorithm to detect and track humans in the scene.

Detected humans are highlighted in **red** in a live Open3D visualiser, while the environment is rendered in **blue**. A live terminal dashboard displays position, distance, velocity, displacement, and total movement for each tracked person.

---

## Features

- Live 3D point cloud visualisation — white background, blue environment, red humans
- Background subtraction using a pre-scanned empty-room map and KD-tree lookup
- RANSAC-based plane removal (floor, ceiling, walls)
- Voxel clustering with BFS neighbour grouping
- Three-classifier voting system — shape, normal, and shadow classifiers
- Multi-human tracking with persistent IDs across frames
- Per-human velocity, displacement, and total distance readout
- Reflection and noise filtering
- Configurable scan radius, detection region, and all tuning parameters

---

## How It Works

### 1. UDP Receiver
The Unitree L2 streams point cloud data over UDP from `192.168.1.62` to port `6201`. The receiver runs in a background thread and feeds raw packets into a queue.

### 2. Packet Parser
Each UDP packet starts with magic bytes `55 AA 05 0A` confirming it is a valid LiDAR packet. The parser extracts calibration parameters and raw range values, applies the full spherical-to-Cartesian coordinate conversion, and outputs XYZ points in metres.

### 3. Background Map
With no people in the room, a 30-second scan is saved as a reference map. During live detection, each frame is compared against this map using a KD-tree — any point within 0.2m of a background point is discarded as static environment. What remains is the foreground: new objects and people.

### 4. Detection Pipeline
Each frame's foreground points pass through the following stages:

| Step | Description |
|------|-------------|
| Region crop | Limits detection to a configurable XYZ bounding box and radius |
| Reflection filter | Removes isolated points with fewer than 3 neighbours within 10cm |
| RANSAC plane removal | Iteratively removes flat surfaces (floor, walls, ceiling) |
| Voxel clustering | Groups points into 10cm voxels, then BFS-connects neighbours into objects |
| Classifier voting | Each cluster is voted on by 3 classifiers — 2/3 votes = human |

### 5. Three-Classifier Voting

**Shape Classifier** — checks if the cluster bounding box matches human dimensions (0.8–2.2m tall, 0.15–0.9m wide).

**Normal Classifier** — runs PCA on the cluster to find the dominant axis. For an upright human, this axis should align with Z (vertical).

**Shadow Classifier** — counts points within 0.5m of the cluster centroid. A real human body returns dense point coverage; reflections and thin objects do not.

### 6. Multi-Human Tracking
Detected human centroids are matched to existing tracks using nearest-neighbour search. Each track maintains a persistent ID, position history, cumulative displacement, and velocity. Tracks that go unmatched for 2 seconds are dropped. Clusters closer than 0.8m are merged to prevent one person being split into multiple detections.

## Requirements

### Hardware
- Unitree L2 4D LiDAR
- Ethernet connection to PC
- LiDAR IP: `192.168.1.62`, PC IP: `192.168.1.2`

### Software
- Python 3.11
- Dependencies:

```
pip install numpy open3d scikit-learn scipy filterpy
```

---

## Setup

### 1. Network Configuration
Set your Ethernet adapter (the one connected to the LiDAR) to a static IP:

| Field | Value |
|-------|-------|
| IP Address | `192.168.1.2` |
| Subnet Mask | `255.255.255.0` |
| Gateway | *(leave blank)* |

### 2. Enable UDP Mode on LiDAR
Open the **Unilidar_2** software, go to UDP Mode, set the destination IP to `192.168.1.2`, and click **Open UDP Port**.

### 3. Create Virtual Environment

```powershell
python -m venv venv
venv\Scripts\activate
pip install numpy open3d scikit-learn scipy filterpy
```

## Usage

### Scan Background Map
Run this once with **no people in the room**. The system scans for 30 seconds and saves the empty-room point cloud.

```powershell
python main.py --scan-bg
```

### Run Live Detection

```powershell
python main.py
```

The Open3D window opens showing the live point cloud. Detected humans are highlighted in red. The terminal shows a live dashboard with position, distance, velocity, and displacement for each person.

### View Saved Background Map

```powershell
python main.py --view-bg
```

### Debug Mode
Prints per-cluster classifier votes every frame — useful for tuning detection parameters.

```powershell
python main.py --debug
```

## Configuration

All parameters are in `config.py`. Key values to tune for your environment:

| Parameter | Default | Description |
|-----------|---------|-------------|
| `MAX_SCAN_RADIUS` | `3.0` m | Ignore everything beyond this distance |
| `BG_TOLERANCE` | `0.2` m | Distance threshold for background subtraction |
| `VOXEL_SIZE` | `0.10` m | Clustering resolution — increase if person splits into multiple detections |
| `VOXEL_NEIGHBOR_FACTOR` | `2.5` | How aggressively voxels are connected — increase to merge body parts |
| `HUMAN_MERGE_RADIUS` | `0.8` m | Detections closer than this are merged into one person |
| `HUMAN_HEIGHT_MIN/MAX` | `0.8–2.2` m | Valid human height range |
| `SHADOW_MIN_PTS` | `50` | Minimum point density to pass shadow classifier |
| `MIN_VOTES` | `2` | Classifiers that must agree for a human detection |

## Known Limitations

- Detection works best with the LiDAR on the floor or a table with clear line of sight
- Highly reflective surfaces (whiteboards, glass) can cause false positives — use `MAX_SCAN_RADIUS` and the detection region in `config.py` to exclude them
- Background map must be rescanned if the LiDAR is moved or the room layout changes significantly
- Partial bodies (person behind a table) may not pass the shape classifier height check
